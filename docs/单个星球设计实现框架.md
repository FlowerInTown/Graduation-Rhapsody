# 行星显示效果
1. 每个行星最多有四张图片，图片index从0到3，固定到一个列表中
2. 播放固定的飞船降落画面后，显示烟尘图片，烟尘散去后（渐隐效果），显示index=0的行星图片
2. 进入行星画面后，屏幕左侧和右侧会分别出现左、右两个箭头，点击可以切换到不同的行星背景图，代码逻辑是index +1 或者 -1，当index < 0时，index设置为index上限-1，反之亦然。

# 行星场景渲染
1. 一共有几类行星，分别如下，这决定了行星基础场景图
    1.1 类地行星
    1.2 冰巨星
    1.3 类地岩石行星
    1.4 矮行星
2. 整个行星探索画面由行星基础场景图 + 背景图构成
    2.1 行星基础场景图是前景，背景图则是背景
    2.2 背景图是由以下两种图片叠加而成
        2.2.1 星空背景图
        2.2.2 大气图，每个行星的大气密度和颜色不一样，具体渲染逻辑参见【行星大气渲染方法】
        2.2.3 根据当前行星的大气密度值N，来决定大气图的透明度，叠加到星空背景图上
3. 场景内的物件
    3.1 NPC
    3.2 物品


# 行星大气渲染方法
1. 基本颜色
    1.1 蓝色 / 蓝白色（很常见，≈30%+）
    1.2 白色 / 灰白（非常常见，≈20%–30%）
    1.3 黄色 / 棕色（较常见，≈15%–25%）
    1.4 红色 / 橙色（中等，≈10%–20%）
    1.5 绿色（极少见，<5%）
2. 颜色分布渲染逻辑
    得到基本颜色后，这个纹理的左上（0,1）、左下(0,0)、右上(1,1)、右下(1,0), 再加一个点(0.5, 0)
    (0.5, 0)的颜色值 = 基本颜色 * 大气密度值N
    左下(0,0)、右下(1,0)的颜色值 = 基本颜色 * 大气密度值N *0.8
    左上（0,1）、右上(1,1)的颜色值 = 基本颜色 * 大气密度值N *0.1
    纹理其他地方的颜色在这几个点做插值



# 行星基本数据结构
{
    atmospheric_density：0.3 //0-1之间的浮点数
    planet_type："类地行星" //分别对应类地行星、冰巨星、类地岩石行星、矮行星
    NPC_name："lala"  //NPC的名字
    atmospheric_color: #AEF5F1 //某个颜色值
    Helium-3_reserves: 5 //氦三储藏量
}


# 行星初始化逻辑
1. 0-1之间随机一个值设置到atmospheric_density
2. 0-10之间随机一个浮点数，设置到 Helium-3_reserves
3. 根据行星大气渲染方法的基本颜色，及其分布概率，随机一个基本颜色值设置到atmospheric_color
4. NPC的数量有限且不重复，假设总行星数量为N，总NPC数量为M，当前行星数据初始化的时候，则有M/N的概率，从NPC列表中抽取一个，NPC列表POP掉这个NPC名字，设置到NPC_name
5. 地行星、冰巨星、类地岩石行星、矮行星四种基本类型中选随机一个，设置到planet_type
